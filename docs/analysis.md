# Analysis

## 1. Problem Statement
Music festival websites often lack interactivity and modern user features. Users struggle to arrange transport, find updates, or connect with other attendees. Organizers require a platform where they can easily manage content without coding.

## 2. Solution Overview
The Running Free Festival platform solves this by:
- Allowing visitors to view festival info.
- Enabling ridesharing between attendees.
- Letting users send messages to coordinate.
- Providing a future CMS for news & articles.
- Supporting secure authentication and payments.

## 3. Stakeholders
| Stakeholder   | Needs |
|---------------|-------|
| Users         | Buy tickets, find rides, chat |
| Drivers       | Offer rides, manage passengers |
| Organizers    | Publish news, manage platform |
| Developers    | Clean architecture, scalable code |
| Artists/Bands | Visibility, promotion on site |

## 4. Constraints
- Deadline set by university.
- Hosting and domain setup required.
- Team experience level varies.
- Must use existing tech stack.

## 5. Technologies Justification
- **FastAPI** → simple, fast Python backend.
- **React + MUI** → modular and responsive UI.
- **MySQL** → relational structure for rides & users.
- **Firebase** → easy, secure user authentication.
- **Stripe** → tested payment solution.
